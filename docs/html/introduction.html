Testing Installation¶

The code has been setup to use setuptools for distribution. code can then be tested using the develop function.

Additionally the applicaiton can be tested in a virtual environment, this is the equivelent of a clean python install, allowing any problems Steps to install the project in a test enviroment are.

Note

If you are installing the base code, you will also need to compile follow the instructons :TODO: here

  1. Install Virtual-Env:

    $sudo apt-get install python-virtualenv

  2. Setup Virtual Environment. Using the –no-site-packages flag means that we get the equivalent of a clean python install:

    $virtualenv --no-site-packages testenv

  3. Activate the virtual env:

    $cd testenv
$source bin/activate

  4. Install the code in development mode:

    $python setup.py develop

Building the Documentation¶

The Documents are built with Sphinx <http://sphinx.pocoo.org/index.html>`_

Most of the information comes from docstrings in the code itself. Please keep these up to date.

To build the Docs it is as simple as:

$cd docs
$make html

Exporters are available for HTML and Latex (amongst others)

Warning

Some Ubuntu versions of sphinx are old, and will generate an error when building the graphs. To aviod this install the latest version of sphinx from PyPy:

#sudo pip install --upgrade Sphinx
